🧹 Refactor checkAfterCredits function

🎯 What:
Extracted the complex HTML fetching, matching, and parsing logic out of `checkAfterCredits` into two new helper functions: `searchAfterCreditsMatch` and `parseAfterCreditsPage`.

💡 Why:
The original function was nearly 150 lines long, performing distinct logical operations including search query generation, HTML list parsing to find a match, specific movie page loading, tag categorizing, and section parsing for content details. Splitting these into logically separated helper functions significantly improves readability, reduces nesting, and makes the code easier to test and maintain without altering behavior.

✅ Verification:
- Validated Javascript syntax using `node -c server.js`.
- Verified that all logic, variable scoping, and the SSRF `validateUrl` checks were perfectly preserved.
- Code review complete and passed. No formal test suite is currently implemented in package.json.

✨ Result:
The codebase is cleaner, `checkAfterCredits` acts as an orchestrator instead of a monolith, and readability and maintainability are substantially enhanced.
